Created by: Frank Stallone III & Katharine Compton

How-to

We made this nifty Markup Guide to help developers through the process of building law firm modular web sites. There is a lot of theory in this Markup Guide but first let's start with a process!

1 The journey begins!

First, review the Folder Structure and Photoshop file provided by the designer. Notice all Layer Comps and Notes given by the designer for direction on intended interactions. Start to think visualize the different modular zones and figure out what your naming conventions will be.

Remember this is a markup guide, so we're going to assume the site is already spooled on WordPress!

3 Cascading Stylesheet Markup

Our HTML is complete for our templates. Now it's time to dive into our Cascading Stylesheets. With our resets, normalize.css, baseline styles and helper classes we have a lot of help to make our code most efficient!

5 WordPress Template

Now we're ready to dive into some PHP! We're using WordPress and have a ton of snippets from making navigations dynamic to utilizing hooks. This is where we upload our template and get everything working on this lovely content management system. =)

2 HyperText Markup

Time to put all the modular zones HTML snippets together for the inner page and home page templates! These clever snippets of code are modeled after the beloved 960 grid framework we use. Again make sure you are sticking to the naming conventions we have setup for this process.

4 JavaScript

It's obvious we've been browser testing as we go along this process, right? With that out of the way it's time to add some some enhancements via JavaScript. We're using jQuery as our main framework, and we have our plugins and scripts well organized!

That's it!
